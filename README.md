# Landing Bot

Телеграм-бот-воронка для найма: кандидат приходит по ссылке → выбирает удобное время созвона → получает напоминания → после созвона автоматически получает контакт куратора.

Технологии: **aiogram 3**, **aiosqlite**, **APScheduler**, деплой в **Docker**.

---

## Быстрый старт (локально)

```bash
cp .env.example .env          # заполнить BOT_TOKEN и ADMIN_IDS
pip install -r requirements.txt
python bot.py
```

### `.env`

```
BOT_TOKEN=<токен от @BotFather>
ADMIN_IDS=<telegram_user_id>[,<id2>,...]   # через запятую, без пробелов
TIMEZONE=Europe/Moscow
DB_PATH=landingbot.db
```

`ADMIN_IDS` — числовые Telegram ID (узнать через @userinfobot). Только они видят `/admin`.

> ⚠️ Файлы `.env*` НЕ коммитятся (см. `.gitignore`). Токен бота — секрет, храните только в `.env`.

---

## Деплой (Docker на VPS)

Скрипт `deploy.sh` копирует исходники на сервер, пересобирает образ и перезапускает контейнер.
Адрес сервера задаётся переменной окружения `REMOTE_HOST`:

```bash
REMOTE_HOST=1.2.3.4 bash deploy.sh
```

На сервере ожидается:
- `/opt/landingbot/` — исходники + `Dockerfile` + `.env`
- `/root/landingbot/docker-compose.yml` + том `./data` для БД

Проверка логов:

```bash
docker logs landingbot --tail 30 -f
```

## Как пользоваться

### Кандидат

1. Переходит по ссылке `t.me/<bot>?start=<source>` (метка источника запоминается).
2. Видит вакансию и кнопки с ближайшими созвонами — выбирает удобное время.
3. Получает ссылку на созвон и своего куратора, подтверждает участие.
4. Получает напоминания: за 1 час, за 15 минут (со ссылкой) и на старте.
5. Через несколько минут после начала созвона автоматически получает контакт куратора.

### HR / Администратор — меню `/admin`

| Пункт | Действие |
|---|---|
| 📅 Назначить созвон | Дата/время (МСК) + ссылка. Инвайты рассылаются автоматически. |
| 📝 Редактировать вакансию | Глобальный текст вакансии. |
| 👥 Кураторы | Список кураторов, распределение кандидатов 50/50, вкл/выкл каждого. |
| 🆘 Контакт для подбора времени | Контакт HR для «ни одно время не подходит». |
| 📊 Статистика | Воронка общая + по созвону + по источнику. |
| 🔗 Реферальные ссылки | Ссылки под каждый канал трафика. |
| 📥 Экспорт в Excel | Все кандидаты со статусами и источником. |
| 📢 Рассылка | Всем или участникам конкретного созвона. |
| 🚫 Чёрный список | Загрузка/выгрузка username. |
| ✏️ Редактировать созвон | Смена времени/ссылки с уведомлением участников. |
| ❌ Отменить созвон | Снимает пуши, уведомляет участников. |

### Логика инвайтов

- Максимум **3 инвайта на кандидата** (`settings.max_invites`).
- При создании созвона инвайты идут новым кандидатам и тем, кто отказался/промолчал — в рамках лимита.

### Кураторы (round-robin)

- Кандидаты распределяются поровну между активными кураторами.
- Кандидат закрепляется за одним куратором на все свои созвоны (sticky).
- Если все кураторы отключены — используется запасной `manager_contact`.

### Пуши

За 1 час, за 15 минут (со ссылкой) и в момент старта — только подтвердившим.
Восстанавливаются автоматически при перезапуске бота.
